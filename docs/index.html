The Agent — What It Built & Where Humans Were Needed

A multi-agent LangGraph pipeline. Each agent has a single responsibility, a structured output schema, and a retry budget. No monolithic "do everything" prompt.

🔍
1. Documentation Finder
Queries Tavily web search twice with different query formulations. Scores each candidate URL by domain match, path structure (/docs, /api, /reference), and cross-query agreement. Selects highest-scoring URL.
✓ No human needed — URL scoring is deterministic
📄
2. Documentation Parser
Calls Firecrawl to extract clean markdown from docs pages. Falls back to Browser Use + Playwright if Firecrawl returns empty (bot-protected pages). Discovers auth/API/pricing subpages and chunks content into 8k-token segments.
⚑ Human needed: 3 apps had login-walled docs (Gladly, DealCloud, Waterfall.io)
⚡
3. Parallel Extraction (×4)
Auth Extractor, API Analyzer, Dev Portal Agent, and MCP Detector run concurrently on the same chunks. Each uses GPT-4o with a structured Pydantic output schema — no free-form text.
⚑ Human needed: access model was ambiguous for Salesforce and WhatsApp — both corrected
🔗
4. Evidence Collector
Creates a sourced Evidence record for every non-null field. Validates that source URLs are HTTPS and reachable. Flags any field with no evidence as confidence=0.0.
✓ Automated — all 100 apps got at least 3 evidence records
✅
5. Verification Agent
Independent second pass using different doc sections and adversarial prompt framing. Compares Pass A vs Pass B field-by-field. Disagreements trigger the Tiebreaker Agent (GPT-4o reads both values).
✓ Human reviewed 12 flagged records — corrected 2
📊
6. Confidence Scorer
Weighted formula: evidence_confidence × agreement_bonus × browser_verification_bonus. App-level score is a weighted average (auth=25%, api=20%, access=20%, mcp=10%, verdict=15%, docs=10%).
✓ Fully automated
💡
7. Pattern Discovery Engine
SQL aggregates over all 100 verified records. Computes auth distribution, API surface distribution, MCP gap metric, easy wins (Self-Serve + REST + no MCP), and per-category breakdowns.
✓ Fully automated
